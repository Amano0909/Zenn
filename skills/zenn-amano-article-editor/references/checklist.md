# Zenn記事チェックリスト

## 文体

- 天野のくだけた一人称の文体を保つ。
- いかにもAIが書いたような無難で均質な文章にしない。
- 読みにくさにつながらない限り、冗談、迷い、個人的な背景は消さない。
- 短く読みやすい段落を優先する。
- 箇条書きにした方が読みやすい箇所はリストにする。

## 誤字脱字と表記

- 明らかな誤字脱字は修正する。
- 製品名が明らかに間違っている場合は、下記の表記にそろえる。
  - WordPress
  - GitHub
  - Zenn
  - Codex
  - Cloudflare Pages
  - Astro
  - Xserver
- 技術用語を勝手に広い意味の言葉へ置き換えない。

## 技術的な正確性

- まずローカルリポジトリの文脈から主張を確認する。
- 最新情報、外部サービスの仕様、ローカルだけで確認できない内容は、必要に応じて調査するかユーザーに確認する。
- 確認できた事実と推測を混ぜない。
- 特に下記は注意して確認する。
  - Zenn CLIのルール
  - ZennのFront Matter制約
  - 画像サイズ制限
  - GitHubリポジトリ容量の目安
  - WordPress XML/WXRの挙動
  - WordPressの画像エクスポートの挙動
  - Cloudflare Pages/Astroのデプロイに関する記述

## Zenn形式

- 記事には下記を含むFront Matterを付ける。
  - `title`
  - `emoji`
  - `type`
  - `topics`
  - `published`
- 詳細な制約はリポジトリの `AGENTS.md` に従う。
- 見出しは原則 `##` 以降を使う。
- コードブロックには、可能な限り言語名を付ける。
- 生HTMLよりも、Zennのメッセージ記法を優先する。
- 最終記事にWordPressショートコードを残さない。

## 記事への変換

- ユーザーがZenn記事化を依頼した場合のみ、`articles/<slug>.md` を作成する。
- ユーザーが明示的に公開を依頼しない限り、`published: false` にする。
- slugはリポジトリのルールを満たすものにする。
- ユーザーが明示的に削除を依頼しない限り、元の下書きは残す。
